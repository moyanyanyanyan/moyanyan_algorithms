# 哈希表总结

## 哈希表一般作用

1.一般来说哈希表都是用来快速判断一个元素是否出现集合里

## 常见的三种哈希结构

### 数组作为哈希表

242.有效的字母异位词 383.赎金信
**核心特点：**是数组的大小是受限的，而且一般用到的不是访问，对字母的处理可以考虑数组，因为每个字母都可以去有方法对应到下标

### set 作为哈希表

349.两个数组的交集
**核心特点：**输出结果中的每个元素一定是唯一的，也就是说输出的结果的去重的， 同时可以不考虑输出结果的顺序

### map 作为哈希表

1.两数之和
**核心特点**可以一次性储存两个值，以键值对形式存储

## 两数之和

1.哈希用来记住遍历过的数组值与数组下标，每次都拿来查询哈希里面有没有对应的值

## 快乐数

1.用 set 来记住快乐数在迭代中的值，然后可以快速查询去判断有没有循环

## 两个数组的交集

1.本来是去进行双重循环，判断是否相等，如果相等去遍历相等数组，如果已经有就不添加，没有就添加 2.把两个数组都转换成 set，遍历一个 set，每次都在另外一个函数里面用判断的 has 方法，做到时间复杂度的大大减少
